barakuda_bringup
===

[[_TOC_]]

This package aims to organise configs and launch files in the easiest way possible.

## Launch files

There are "two" types of launch files:

- The ones that are supposed to be launched. Those launch files end in `.launch`.
- Those that are modules/templates that are supposed to be included in other launch files. Those launch files end in
  `.launch.xml`.

Typically, launch files supposed to be launched are for robots (e.g. `taurus.launch`).

### How to add a new robot/new config for a robot

The best way to add a new robot or config for a robot is to include the `base.launch.xml` after settings all the
available arguments

## Config files

Except for a few configuration files that are defined with a full path (and passed through as arguments in
`base.launch.xml` — see the previous section), the paths of the configuration files are defined using the robot_name
argument. The configurations will be searched for in `$(find barakuda_bringup)/config/$(arg robot_name)/`. For instance,
the `sbg.yaml` configuration file should be placed at `$(find barakuda_bringup)/config/$(arg robot_name)/sbg.yaml`.

## URDF

URDF are fetched from [barakuda_description](https://gitlab.ensta.fr/ssh/barakuda_description). Look at the README there
for more details.
